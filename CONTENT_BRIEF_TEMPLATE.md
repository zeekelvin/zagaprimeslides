# ZagaPrime Content Brief Template

Use this exact structure when you want new branded content generated consistently.

```text
Create a new ZagaPrime carousel set.

Collection slug: [short-url-slug]
Collection title: [public title]
Goal: [what this content should make the reader think, feel, or do]
Audience: [who this is for]
Content type: [tutorial | myth-buster | tools list | founder lesson | lead magnet teaser]
Topic: [single topic]
Core promise: [single transformation]
CTA keyword: [ONE WORD]

Brand voice:
- direct
- credible
- street-smart
- practical
- no corporate filler

Design system:
- use the ZagaPrime / Knox-inspired style already defined in `SKILL.md`
- 1080x1080 PNG slides
- 10-slide structure
- no visible "HOOK" or "CTA" labels

Slide direction:
1. Cover
2. [custom label for slide 2]
3. Problem
4. Cost
5. Shift
6. System
7. Results
8. Decision
9. Moment
10. Close

Offer details:
- link or resource being promoted: [optional]
- landing page copy needed: [yes/no]
- captions package needed: [yes/no]

Source points:
- [fact, lesson, workflow, or offer detail]
- [fact, lesson, workflow, or offer detail]
- [fact, lesson, workflow, or offer detail]

Constraints:
- [words or claims to avoid]
- [required phrasing or offer details]
```

## Minimal Prompt

```text
Using the ZagaPrime carousel creator skill and the repo structure in `zagaprime-hub`, create a new collection about [topic] for [audience]. The promise is [transformation]. CTA keyword is [WORD]. Build the slides, add the collection to the hub, and regenerate the hub pages.
```

## Best Practice

- Keep each collection focused on one belief, one workflow, or one lead magnet.
- Provide the CTA keyword up front so the close slide and caption package stay aligned.
- If you want a guide page too, explicitly ask for `index.html` content for that collection.
- If the content is opinionated or technical, include your raw talking points so the copy keeps your voice.
